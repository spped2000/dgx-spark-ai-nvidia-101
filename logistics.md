### 5.2 Learning Outcomes (จบคอร์สแล้วคุณจะ...)

หลังเรียนจบ 1 วัน ผู้เรียนจะสามารถ (อธิบายแบบมือใหม่ + โยงเข้าแกนคอร์ส):

1. **อธิบายได้ว่า DGX Spark คืออะไรและทำไมพิเศษ** — เครื่อง AI ระดับ desktop (GB10 Grace Blackwell, 128GB unified memory) ที่ใช้ "หน่วยความจำก้อนเดียว" และเข้าใจว่าคอขวดจริงคือ **memory bandwidth** ไม่ใช่ความจุ
2. **ใช้ LLM บนเครื่องผ่านเบราว์เซอร์ได้เอง** — ต่อเข้า **shared endpoint** เดียว แชตกับโมเดล (เช่น Nemotron Nano) และเข้าใจว่าทำไมทั้งห้อง 30 คนถึงใช้เครื่องเดียวพร้อมกันได้ (continuous batching)
3. **เข้าใจ Pillar 1 — Optimize Tokens** — รู้ว่า **"cost per token = KPI ใหม่"** และเห็นผลของ **prefix/KV caching**, **quantization (FP8/NVFP4)** กับความเร็ว/ค่าใช้จ่ายจริง พร้อมแนวคิด **Context Rot** ("window ใหญ่ ≠ สมองใหญ่")
4. **เข้าใจ Pillar 2 — Harnessing** — มองออกว่า **"Agent = Model + Harness"** (ลูป `CONTEXT → OBSERVE → REASON → ACT`) และเห็น **multi-agent** (orchestrator สั่ง sub-agent) + การใช้ tool ผ่าน **MCP** ทำงานจริง
5. **เข้าใจ Pillar 3 — Secure & Sandbox** — เห็นด้วยตาว่า **OpenShell** (ALPHA) บล็อก agent ออกเน็ต (403), แก้ **YAML policy** เปิดสิทธิ์แบบ read-only, และอ่าน **audit trail** ได้ → เข้าใจหลัก **deny-by-default + signed skills**
6. **รู้ว่าจะไปต่อทางไหน** — รู้จัก ecosystem (vLLM, NIM, Triton, TensorRT-LLM, NeMo, Nemotron family) และมี playbook อ้างอิง (build.nvidia.com, github.com/NVIDIA/dgx-spark-playbooks, huggingface.co/nvidia) ไว้ฝึกต่อ

> **หมายเหตุความถูกต้อง:** คอร์สนี้เป็นระดับ **เริ่มต้น/ปูพื้น (no-code)** — outcome เน้น "เข้าใจ + ลงมือเห็นผล" ไม่ใช่ "เขียน production system ได้เอง" และ **ไม่ใช่** หลักสูตรที่ออก certification ทางการของ NVIDIA
